# doc-javadoc-public-api

> Document all public API with Javadoc

## Why It Matters

Public classes and methods are a contract with every future caller, including your future self six months from now, and Javadoc is the only documentation format that IDEs render inline on hover and that tools like `javadoc`/Dokka-equivalent site generators can publish automatically. Undocumented public API forces every caller to read the implementation to understand intent, preconditions, and edge cases - work that should have been done once by the author instead of repeatedly by every consumer.

## Bad

```java
public class RateLimiter {

    // No Javadoc - callers must read the implementation to learn the contract
    public boolean tryAcquire(String clientId, int permits) {
        Bucket bucket = buckets.computeIfAbsent(clientId, id -> new Bucket(capacity));
        return bucket.tryConsume(permits);
    }

    public void reset(String clientId) {
        buckets.remove(clientId);
    }
}
```

## Good

```java
/**
 * Limits the rate of operations per client using a token-bucket algorithm.
 *
 * <p>Each client identified by a unique key gets an independent bucket that
 * refills at a fixed rate. Instances are safe for concurrent use by multiple
 * threads.
 */
public class RateLimiter {

    /**
     * Attempts to acquire the given number of permits for a client without blocking.
     *
     * @param clientId unique identifier of the calling client; must not be null
     * @param permits number of permits to acquire; must be positive
     * @return {@code true} if the permits were acquired, {@code false} if the
     *     client's bucket did not have enough permits available
     * @throws IllegalArgumentException if {@code permits} is not positive
     */
    public boolean tryAcquire(String clientId, int permits) {
        Bucket bucket = buckets.computeIfAbsent(clientId, id -> new Bucket(capacity));
        return bucket.tryConsume(permits);
    }

    /**
     * Discards the bucket state for a client, effectively resetting its rate limit.
     *
     * @param clientId unique identifier of the client to reset; must not be null
     */
    public void reset(String clientId) {
        buckets.remove(clientId);
    }
}
```

## What Does Not Need Javadoc

Private and package-private members generally do not need Javadoc unless the logic is non-obvious; the cost of documenting every implementation detail outweighs the benefit, and the Google Java Style Guide only mandates Javadoc for members exposed by a class's public API. Trivial, self-explanatory public getters/setters generated by records or simple accessors are also commonly exempted in practice, though documenting the class itself is still expected.

```java
class Bucket {
    // Package-private helper, not part of the public API - Javadoc optional
    boolean tryConsume(int permits) {
        // ...
        return true;
    }
}
```
